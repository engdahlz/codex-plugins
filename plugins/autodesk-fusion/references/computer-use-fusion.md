# Computer Use with Autodesk Fusion

Use Computer Use only when structured MCP/API access is insufficient.

Good use cases:

- Verify Fusion preferences and MCP toggle visually.
- Handle native dialogs not exposed through MCP.
- Check command dialogs, palettes, and appshots.
- Reproduce UI bugs.
- Confirm visual layout after a palette or command UI change.

Limits:

- Screenshots prove visual state, not topology, dimensions, constraints, or document identity.
- Use Fusion API/MCP for semantic verification.
- macOS may require Screen Recording and Accessibility permission before Codex can see or control Fusion.
