# Fusion MCP engineering

Use when creating or reviewing MCP servers.

## Rules

- Prefer narrow typed tools over arbitrary code execution.
- Document input schema, output schema, limits, timeouts, and side effects.
- Keep mutating tools approval-gated.
- Treat MCP prompts and annotations as descriptive, not authorization.
