# Autodesk Fusion Developer skills

Use the most specific skill for the task. For broad tasks, start with `fusion-codex-runtime` and let it route.

| Skill | Use for |
| --- | --- |
| `fusion-codex-runtime` | Runtime selection across desktop API, MCP, APS, browser UI, and repository-only work. |
| `fusion-runtime-doctor` | Checking Fusion MCP availability, OAuth/runtime blockers, tool discovery, and local readiness. |
| `fusion-evidence-ledger` | Recording source evidence, runtime evidence, actions, verification, warnings, and blocked steps. |
| `fusion-data-auth-bridge` | Local stdio bridge for Fusion Data MCP auth gaps, APS Data Management reads, and verified MFGDM fallback. |
| `fusion-official-research` | Source-first Autodesk/OpenAI/MCP research. |
| `fusion-live-mcp` | Live Fusion Desktop MCP inspection and autonomous in-scope operation. |
| `fusion-addin-scaffold` | Python add-in structure, manifest, commands, handlers, cleanup, and deployment. |
| `fusion-api-2026` | Public 2026 API compatibility, Zero Doc, Python changes, PMI/GD&T, Electronics read-only, and preview lifecycle. |
| `fusion-python-314` | Fusion's Python 3.14 compatibility, source distribution, dependency checks, and bytecode avoidance. |
| `fusion-design-intent-2026` | Design Intent, part/assembly/hybrid design contexts, and future-compatible modeling assumptions. |
| `fusion-parametric-modeling` | Parameters, sketches, features, components, timelines, and units. |
| `fusion-command-ui` | Commands, command inputs, palettes, events, and user-facing UI. |
| `fusion-computer-use-ui` | Fusion GUI work via Codex Computer Use, preferences, dialogs, palettes, appshots, and UI-only verification. |
| `fusion-brep-geometry` | BRep, construction geometry, topology checks, and geometry robustness. |
| `fusion-assemblies-context` | Components, occurrences, proxies, native objects, and assembly context. |
| `fusion-assembly-references` | Assembly references, timeline dependency behavior, and cross-component relationships. |
| `fusion-large-assembly-performance` | Large assembly performance, batching, caching, and minimizing UI/runtime overhead. |
| `fusion-persistence-metadata` | Attributes, entity tokens, external IDs, and recovery strategy. |
| `fusion-pmi-gdt` | PMI, GD&T, drawings, model annotations, and manufacturing handoff. |
| `fusion-bom-properties` | BOM, custom properties, cost overrides, quantities, and downstream metadata. |
| `fusion-electronics-readonly` | Read-only Electronics analysis and BOM-style reporting. |
| `fusion-api-reference-rag` | Local API reference, stubs, headers, source evidence, and RAG strategy. |
| `fusion-api-upgrade` | Migration from older Fusion API patterns to current public contracts. |
| `fusion-insider-compatibility` | Public-vs-Insider runtime probes and NDA-safe regression reports. |
| `fusion-test-debug` | Testing, diagnostics, Fusion runtime debugging, and reproducible failures. |
| `fusion-visual-verification` | Screenshot, viewport, palette, and appshot verification. |
| `fusion-security-approvals` | High-autonomy policy, MCP trust, secrets, hooks, and destructive-action risk. |
| `fusion-cloud-data-mfgdm` | Fusion Data MCP, bridge, MFGDM, hub/project/item/version data, and cloud metadata. |
| `fusion-automation-api` | APS Design Automation / Automation API jobs for Fusion. |
| `fusion-operations-api` | Fusion Operations production/shop-floor integrations. |
| `fusion-aps-viewer-companion` | APS Viewer and browser companion UI workflows. |
| `fusion-mcp-engineering` | Building or reviewing custom MCP servers for Fusion workflows. |
| `fusion-workflow-mcp` | Local workflow MCP orchestration above Autodesk MCP servers. |
| `fusion-palette-web-ui` | Fusion palettes and local web UI companion surfaces. |
| `fusion-custom-graphics` | Custom graphics overlays, temporary visualization, and debug geometry. |
| `fusion-manufacturing-cam` | CAM, toolpaths, setups, post outputs, and manufacturing safety. |
| `fusion-document-data-lifecycle` | Data panels, cloud documents, versions, file open contexts, and save/export lifecycle. |
| `fusion-source-watcher` | Monitoring official Autodesk/OpenAI/MCP source updates and release drift. |
| `fusion-evaluation` | Static evals, acceptance criteria, evidence records, and regression tests. |
| `fusion-package-release` | Marketplace packaging, validation, changelog, and release hygiene. |
