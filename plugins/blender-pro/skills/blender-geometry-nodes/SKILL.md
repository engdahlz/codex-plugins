# Blender Geometry Nodes

## Purpose

Help Codex design, review, and improve Geometry Nodes systems in Blender.

## Workflow

1. Clarify the procedural goal: scatter, array, growth, deformation, layout, terrain, cables, shelves, panels, particles, or asset variations.
2. Inspect existing node groups and modifiers when MCP access exists.
3. Define input parameters before building the graph.
4. Keep the node group name descriptive and expose useful controls.
5. Prefer stable, readable node networks over one opaque mega-graph.
6. Add object-level naming and collection organization around generated results.
7. Verify scale, density, instance count, viewport performance, and final render behavior.

## Good outputs

- Parameter list.
- Node group structure.
- Expected object/collection names.
- Performance concerns.
- Verification checklist.
